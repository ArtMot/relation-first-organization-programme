# Application Fast-Follow Queue

**Status:** planning/control document for the first post-footprint application release.  
**Recommendation:** keep the first Zenodo DOI focused on the foundation/manuscript footprint; publish applications as a fast-follow release after the initial DOI exists.

## Rationale

The first public Zenodo release should establish the programme spine, manuscript paths, non-claims, and citation layer. Adding application papers before that release risks mixing three different messages:

1. foundation/programme footprint;
2. externally publishable mathematical exemplars;
3. frontier empirical or physical branch claims.

The better sequence is:

```text
v0.1 = public manuscript footprint / foundation corpus
v0.2 = application and exemplar release I
v0.3 = frontier branch release, if branch maps and claim controls are ready
```

This keeps the first DOI clean while allowing fast public follow-up.

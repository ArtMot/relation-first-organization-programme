Recursive Bridge Calibration is a central methodology paper. It bridges the earlier foundation/no-go stack into the later discovery-methodology layer, including Bridge-Valence Diagnostics and Generative Bridge Search. It should remain in the main public corpus, but it is not yet cleanly public-PDF-ready.

The content is important and should not be compressed away. The suitability issue is register: the abstract and opening still contain process/evolution language and programme-internal planning language. These should be converted into stable paper structure before public release.

## Main suitability issues

### 1. Title needs public subtitle

Recommended title:

```text
Recursive Bridge Calibration: Calibrators, Residuals, and Bridge Selection in Relation-First Inference
```

### 2. Draft subtitle should move to changelog

Current front matter says:

```text
Draft v1.7 — typed micro-triad diagnostics and local upper-triad procedure
```

Move this to changelog/provenance before public PDF release.

### 3. Abstract is still too process-heavy

The abstract says the procedure was implicit and made explicit during the Lorentzian bridge chain, then lists branch-atlas paths. That is useful internally, but the public abstract should present the method as stable structure:

```text
This paper defines Recursive Bridge Calibration, a method for selecting and revising bridge candidates under declared source, target, boundary, readout, calibrator, obstruction, and trace-update data.
```

### 4. Lorentzian chain should be example, not anchor

The Lorentzian bridge chain can remain the main worked example, but the paper's public identity should be method-first, not Lorentzian-chain-first.

### 5. Branching atlas may overload public release

The branch atlas of outward paths is valuable, but it may read like a speculative programme roadmap if placed too early. Consider moving detailed branch-atlas material to an appendix or public reader map.

### 6. Numbering issue spotted

The inspected text has Section `# 3. Primitive schema` followed by subsections labelled `## 2.1`, `## 2.2`, `## 2.3`. Fix numbering before PDF release.

### 7. Typographic issue spotted

The inspected text includes `\(\v\v\varepsilon_V\)` in the boundary examples. Check whether this is a typo for `\varepsilon_V`.

# Public Title Register

This register separates internal file names from public-facing manuscript titles. Some RFOP filenames are useful for internal provenance but too esoteric for public release, arXiv, or journal submission.

## Title policy

A public title should:

1. identify the paper's actual contribution;
2. preserve the programme's technical meaning;
3. avoid sounding like a private code name;
4. avoid overclaiming physical or universal results;
5. move very dense terminology into a subtitle where useful;
6. distinguish foundation papers from external-facing exemplar/application papers.

## BVD / GBS special note

Bridge-Valence Diagnostics and Generative Bridge Search are core methodology papers. Their titles must be legible enough for outside readers because they explain how RFOP enters frontier problems in a controlled way.

The recommended public pair is:

```text
Bridge-Valence Diagnostics: A Relation-First Method for Classifying Bridge Problems
```

and:

```text
Generative Bridge Search: A Relation-First Method for Producing Candidate Bridge Solutions
```

Subtitles may preserve the dense technical classification machinery.

## Rule

Do not change filenames just to improve titles. Filenames preserve provenance. Public titles belong in manuscript front matter, reader maps, release notes, and citation metadata.

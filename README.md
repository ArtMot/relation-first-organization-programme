## 1. Purpose

This document is the public navigation layer for the Relation-First Organization Programme (RFOP). It is designed to make the programme legible as a coherent research corpus before individual article submissions fragment the context.

The current purpose is not to compress the programme into a slogan and not to replace the full papers. The purpose is to show:

1. what the programme is;
2. which documents form the public canonical spine;
3. which documents support external review and reproducibility;
4. which branches are active frontier branches;
5. which items are held, internal, or superseded;
6. how the publication and public-footprint strategy should proceed.

## 2. One-paragraph programme statement

The Relation-First Organization Programme is a mathematical and methodological research programme in which organization, observation, learning, objecthood, residuals, bridge construction, and physical emergence are treated as declared relations with explicit boundary, mediation, readout, task, and calibration structure. Its foundation defines proper organization through quotient-presented resolution systems; charting makes organization observer-relative; recursive chart learning treats admitted path traces as learning; triadic closure explains why boundary, mediation, and visibility form the first closure-complete role schema; No Undeclared Relation blocks hidden transfer; Recursive Bridge Calibration supplies bridge-selection and calibration discipline; Bridge-Valence Diagnostics classifies the type of bridge problem being faced; and Generative Bridge Search turns diagnosed question-shapes into candidate-answer-shapes under typed matching, adaptation, guardrails, and verification obligations.

## 3. What v2.0 changes

The v2.0 baseline is materially different from the earlier 25-paper release plan.

The programme now has a core discovery-methodology layer:

```text
No Undeclared Relation
-> Recursive Bridge Calibration
-> Bridge-Valence Diagnostics
-> Generative Bridge Search
-> controlled frontier bridge construction
```

This matters because the programme is no longer only a foundation stack plus branch papers. It now has a programme-native method for diagnosing bridge problems and generating candidate solution paths without relying on uncontrolled analogy or hidden transfer.

Bridge-Valence Diagnostics and Generative Bridge Search are therefore classified as **main public v2.0 corpus** papers, not as internal audit traces.

## 3A. What v2.1 adds

The v2.1 shell adds a formal version-lineage/provenance layer. The Drive folder now contains prior versions, self-contained variants, and `FULL_PRESERVATION_REPAIR` variants for several paper families, especially the AS / critical-rank branch.

This is handled by a new rule:

```text
Latest version wins only after provenance audit.
```

The main public corpus remains clean. Prior versions and repair variants are not placed in the reader path. They are preserved through `VERSION_LINEAGE_NOTES.md` and the files in `version_lineage/`.

The AS / critical-rank branch is treated as a serious frontier branch requiring its own branch reader map and carry-forward audit before public promotion.

## 4. Non-claims and scope discipline

The programme does **not** currently claim to derive General Relativity, the Standard Model, QED, physical constants, quantum mechanics, empirical dark-sector corrections, or a completed theory of nature.

It also does not claim that:

- recoverability implies causality;
- information geometry alone supplies Lorentzian signature;
- local charted success implies global objecthood;
- residuals automatically become dynamics;
- a generated candidate is a verified solution;
- bridge-calibration guarantees truth;
- the branch atlas is complete.

The honest current claim is narrower and stronger:

> Foundational transfer claims become inspectable only when source, bridge, target distinction, boundary, readout, task, calibrator, and remaining residual are declared.

## 17. Final orientation statement

RFOP v2.0 should be understood as a programme with four public layers:

```text
foundation -> bridge discipline -> discovery methodology -> frontier branches
```

The foundation defines relation-first organization. The bridge discipline blocks hidden transfer. The discovery methodology classifies bridge problems and generates candidate answer-shapes. The frontier branches apply that machinery to Lorentzian, continuum, AS, empirical, and physics-facing targets under explicit non-claims.
